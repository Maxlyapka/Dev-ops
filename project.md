# Комплексний DevOps проєкт: Інфраструктура, Моніторинг та Автоматизація

## 1. Інсталяція Linux та налаштування ОС
* Встановлено три різні дистрибутиви з різними схемами розмітки: Ubuntu (Standard), Rocky Linux (LVM) та Debian (Software RAID1).
* На Rocky Linux створено групу томів (`vg_mysql`) та логічний том (`lv_db`) для гнучкого управління базою даних.
* На Debian налаштовано програмний RAID1 (md0) для кореневого каталогу.

## 2. Мережа та Безпека (SSH, NAT, sudo)
* Змінено стандартний порт SSH на 2222. Вирішено конфлікт із механізмом Socket Activation у нових версіях Ubuntu шляхом зупинки `ssh.socket` та запуску класичного `ssh.service`.
* Налаштовано автентифікацію виключно за ключами `ed25519` та вимкнено доступ за паролем (`PasswordAuthentication no`).
* Створено групу `devops_team`, налаштовано права через `sudoers` для безпечного адміністрування системою.
* Налаштовано NAT (Network Address Translation) через `iptables` для забезпечення доступу внутрішніх нод до мережі Інтернет.
* Розгорнуто власну службу DNS (Bind9) у режимі caching nameserver з пересиланням запитів.

## 3. Відновлення системи (GRUB & Troubleshooting)
* **Сімейство Debian:** Продемонстровано відновлення доступу до адміністративного запису шляхом зміни параметрів завантаження ядра (`init=/bin/bash`).
* **Сімейство RHEL:** Виконано скидання пароля через `rd.break` та відновлення контекстів безпеки SELinux за допомогою файлу `/.autorelabel`.
* **Відновлення UEFI:** Успішно відновлено систему з EFI Shell за допомогою Live CD, підготовки середовища `chroot` та перевстановлення ядра з GRUB.

## 4. Автоматизація резервного копіювання (Bash & Cron)
* Розроблено Bash-скрипт для архівації каталогів `/etc` та `/home` з маркуванням файлів за поточною датою та часом.
* Скрипт автоматично передає архіви на віддалений сервер за допомогою захищеного протоколу SCP.
* Налаштовано планувальник завдань `cron` для щоденного виконання бекапів о 03:00 ночі.

## 5. Веб-інфраструктура (LEMP, Load Balancing, NFS)
* **Load Balancing:** Nginx налаштовано як Reverse Proxy з балансуванням навантаження (використано механізм `upstream`) між двома бекенд-серверами на базі Apache.
* **Shared Storage:** Розгорнуто NFS-сервер для спільної директорії та синхронізації веб-контенту між нодами кластера.
* **SSL:** Налаштовано HTTPS за допомогою сертифікатів Let's Encrypt (`certbot`) з автоматичним перенаправленням трафіку.
* **PHP:** Реалізовано дві схеми обробки: через модуль Apache (`mod_php`) та окремо через `PHP-FPM`.

## 6. Бази даних (MySQL/MariaDB)
* Встановлено та захищено сервер MariaDB (перевірено, що порт 3306 слухається тільки на локальному інтерфейсі 127.0.0.1).
* Налаштовано автоматизований безпарольний вхід для адміністратора через конфігураційний файл `.my.cnf` з безпечними правами доступу 600.
* Виконано аналіз та тюнінг продуктивності бази даних за допомогою скрипта `MySQL Tuner`.

## 7. Моніторинг (Zabbix) та Логування (Graylog)
* **Zabbix 7.0 LTS:** Розгорнуто систему моніторингу, налаштовано Zabbix-агентів на всіх нодах.Виявлено та вирішено проблеми зі збором метрик від Nginx та Apache.
* **Graylog:** Розгорнуто стек централізованого логування (Graylog + OpenSearch + MongoDB) за Nginx Reverse Proxy з SSL-шифруванням.
* Налаштовано прийом логів через Syslog (UDP) на порту 1514 та налаштовано клієнтів через `rsyslog`.
* *Troubleshooting:* Під час інтеграції Graylog зафіксовано нестачу оперативної пам'яті (OOM) через ресурсомісткі Java-процеси, що стало практичним кейсом аналізу ресурсних лімітів сервера.
