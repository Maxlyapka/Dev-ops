# Комплексний DevOps проєкт: Інфраструктура, Моніторинг та Автоматизація

## 1. Інсталяція Linux та налаштування ОС
* Встановлено три різні дистрибутиви з різними схемами розмітки: Ubuntu (Standard), Rocky Linux (LVM) та Debian (Software RAID1).
* На Rocky Linux створено групу томів (`vg_mysql`) та логічний том (`lv_db`) для гнучкого управління сховищем бази даних.
* На Debian налаштовано програмний RAID1 (`md0`) для забезпечення відмовостійкості системи.

## 2. Мережа та Безпека (SSH, NAT, sudo)
* Змінено стандартний порт SSH на `2222`. Вирішено конфлікт із механізмом Socket Activation у нових версіях Ubuntu.
* Налаштовано автентифікацію виключно за ключами `ed25519` та повністю вимкнено доступ за паролем.
* Створено групу `devops_team`, налаштовано права через `sudoers` (принцип найменших привілеїв).
* Налаштовано NAT через `iptables` для забезпечення доступу внутрішніх нод кластера до мережі Інтернет.
* Розгорнуто власну службу DNS (Bind9) у режимі caching nameserver.

## 3. Відновлення системи (GRUB & Troubleshooting)
* Продемонстровано відновлення доступу до адміністративного запису шляхом зміни параметрів завантаження ядра (`rd.break`).
* Успішно відновлено працездатність системи з EFI Shell за допомогою Live CD та середовища `chroot` для перевстановлення завантажувача.

## 4. Автоматизація резервного копіювання (Bash & Cron)
* Розроблено Bash-скрипт для архівації каталогів `/etc` та `/home` з автоматичним маркуванням файлів за датою та часом.
* Скрипт реалізує безпечну передачу архівів на віддалений сервер за допомогою SCP.
* Налаштовано планувальник завдань `cron` для щоденного виконання бекапів о 03:00 ночі.

## 5. Веб-інфраструктура (Nginx, Load Balancing, NFS)
* Nginx налаштовано як Reverse Proxy (Layer 7) з балансуванням навантаження між бекенд-серверами Apache.
* Розгорнуто NFS-сервер для створення спільної директорії та синхронізації веб-контенту між усіма вузлами кластера.
* Налаштовано HTTPS та оптимізовано роботу PHP через PHP-FPM.

## 6. Бази даних (MariaDB)
* Встановлено та захищено сервер MariaDB (виконано `mysql_secure_installation`).
* Налаштовано мережевий доступ для внутрішніх нод кластера.
* Налаштовано автоматизований безпарольний вхід для адміністратора через конфігураційний файл `.my.cnf` з правами доступу `600` для безпеки автоматизації.

## 7. Моніторинг (Zabbix) та Логування (Graylog)
* Розгорнуто систему моніторингу Zabbix 7.0 LTS, налаштовано агентів для збору метрик з Nginx та Apache.
* Розгорнуто стек централізованого логування (Graylog + OpenSearch + MongoDB).
* Вирішено проблему з нестачею оперативної пам'яті (OOM) шляхом аналізу та лімітування ресурсів JVM для Java-процесів під час роботи Graylog.
