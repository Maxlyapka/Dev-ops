# Комплексний DevOps проєкт: Інфраструктура, Моніторинг та Автоматизація

## 1. Інсталяція Linux та налаштування ОС
* Встановлено три різні дистрибутиви з різними схемами розмітки: Ubuntu (Standard), Rocky Linux (LVM) та Debian (Software RAID1).
* На Rocky Linux створено групу томів (vg_mysql) та логічний том (lv_db) для гнучкого управління базою даних.
* На Debian налаштовано програмний RAID1 (md0) для кореневого каталогу.

## 2. Мережа та Безпека (SSH, NAT, sudo)
* Змінено стандартний порт SSH на 2222. Вирішено конфлікт із механізмом Socket Activation у нових версіях Ubuntu.
* Налаштовано автентифікацію виключно за ключами ed25519 та вимкнено доступ за паролем.
* Створено групу devops_team, налаштовано права через sudoers.
* Налаштовано NAT через iptables для забезпечення доступу внутрішніх нод до Інтернету.
* Розгорнуто власну службу DNS (Bind9) у режимі caching nameserver.

## 3. Відновлення системи (GRUB & Troubleshooting)
* Продемонстровано відновлення доступу до адміністративного запису шляхом зміни параметрів завантаження ядра.
* Виконано скидання пароля через rd.break та відновлення контекстів безпеки SELinux.
* Успішно відновлено систему з EFI Shell за допомогою Live CD та середовища chroot.

## 4. Автоматизація резервного копіювання (Bash & Cron)
* Розроблено Bash-скрипт для архівації каталогів /etc та /home з маркуванням файлів за датою.
* Скрипт автоматично передає архіви на віддалений сервер за допомогою SCP.
* Налаштовано планувальник завдань cron для щоденного виконання бекапів о 03:00 ночі.

## 5. Веб-інфраструктура (LEMP, Load Balancing, NFS)
* Nginx налаштовано як Reverse Proxy з балансуванням навантаження між двома бекенд-серверами Apache.
* Розгорнуто NFS-сервер для спільної директорії та синхронізації веб-контенту.
* Налаштовано HTTPS за допомогою сертифікатів Let's Encrypt.

## 6. Бази даних (MySQL/MariaDB)
* Встановлено та захищено сервер MariaDB.
* Налаштовано автоматизований безпарольний вхід для адміністратора через конфігураційний файл .my.cnf з правами доступу 600.
* Виконано аналіз продуктивності бази даних за допомогою MySQL Tuner.

## 7. Моніторинг (Zabbix) та Логування (Graylog)
* Розгорнуто систему моніторингу Zabbix 7.0 LTS, налаштовано агентів на всіх нодах.
* Розгорнуто стек централізованого логування (Graylog + OpenSearch + MongoDB).
* Налаштовано прийом логів через Syslog (UDP) на порту 1514.
* Вирішено проблему з нестачею оперативної пам'яті (OOM) через ресурсомісткі Java-процеси під час роботи Graylog.
